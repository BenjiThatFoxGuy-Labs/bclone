# {{< icon "fas fa-memory" >}} Tmpfs

The tmpfs backend is an ephemeral adapter. It wraps another rclone
remote and treats the wrapped root as disposable temporary storage.

Tmpfs can wrap an in-memory remote, a local directory on a tmpfs
mount, or another backend rooted at a dedicated temporary path. It can
expire old objects by modification time and enforce a total size limit.

**Important:** the wrapped root belongs to tmpfs. Tmpfs may delete
everything under that root on startup, shutdown, or cleanup. Do not
point `tmpfs-remote` at shared or permanent data.

Tmpfs is supported on Unix-like systems in v1. Windows support is
deferred.

## Examples

List a temporary in-memory root:

```sh
rclone lsf :tmpfs,remote=':memory:cli-tmp':
```

Use a size limit with another backend root:

```sh
rclone copy ./work :tmpfs,remote='remote:tmp/work',max_size=4G:
```

## Docker volume examples

Wrap a memory backend bucket:

```sh
docker volume create -d rclone \
  -o type=tmpfs \
  -o tmpfs-remote=':memory:docker-tmp' \
  -o tmpfs-max-size=4G \
  -o tmpfs-max-age=1h \
  docker-tmp
```

Wrap a local directory on a host tmpfs mount:

```sh
docker volume create -d rclone \
  -o type=tmpfs \
  -o tmpfs-remote='/tmpfs/docker-tmp' \
  -o tmpfs-max-size=20G \
  docker-tmp
```

Wrap another backend at a dedicated temporary root:

```sh
docker volume create -d rclone \
  -o type=tmpfs \
  -o tmpfs-remote='remote:tmp/docker-tmp' \
  -o tmpfs-max-age=24h \
  docker-tmp
```

For local paths, tmpfs refuses broad roots such as `/`, `/tmp`,
`/tmpfs`, `/var`, home directories, and Docker's own volume roots.
Use a dedicated child path such as `/tmpfs/docker-tmp`.

### Modification times and hashes

Tmpfs passes modification times and hashes through from the wrapped
remote. Expiry uses object modification time in v1.

### Standard options

Here are the Standard options specific to tmpfs (Ephemeral adapter for a dedicated temporary remote).

#### --tmpfs-remote

Dedicated temporary remote to wrap.

Properties:

- Config:      remote
- Env Var:     RCLONE_TMPFS_REMOTE
- Type:        string
- Required:    true

### Advanced options

Here are the Advanced options specific to tmpfs (Ephemeral adapter for a dedicated temporary remote).

#### --tmpfs-max-size

Maximum total size of data stored in the wrapped root.

Properties:

- Config:      max_size
- Env Var:     RCLONE_TMPFS_MAX_SIZE
- Type:        SizeSuffix
- Default:     off

#### --tmpfs-max-age

Expire objects older than this by modification time.

Properties:

- Config:      max_age
- Env Var:     RCLONE_TMPFS_MAX_AGE
- Type:        Duration
- Default:     off

#### --tmpfs-cleanup-interval

How often to remove expired objects.

Properties:

- Config:      cleanup_interval
- Env Var:     RCLONE_TMPFS_CLEANUP_INTERVAL
- Type:        Duration
- Default:     1m0s

#### --tmpfs-cleanup-on-shutdown

Purge the wrapped root when tmpfs shuts down.

Properties:

- Config:      cleanup_on_shutdown
- Env Var:     RCLONE_TMPFS_CLEANUP_ON_SHUTDOWN
- Type:        bool
- Default:     true

#### --tmpfs-purge-on-start

Purge the wrapped root when tmpfs starts.

Properties:

- Config:      purge_on_start
- Env Var:     RCLONE_TMPFS_PURGE_ON_START
- Type:        bool
- Default:     true
